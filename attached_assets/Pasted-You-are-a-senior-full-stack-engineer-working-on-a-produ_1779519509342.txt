You are a senior full-stack engineer working on a production-grade Next.js 14 + TypeScript + Tailwind CSS + Prisma application for a mobile-first travel platform.

Implement a modern unread notification indicator system similar to iPhone/iMessage behavior.

The goal is to show small unread badges/dots when:

* a user receives a new message
* someone replies to their trip post
* a new group trip message arrives

The indicator should automatically disappear once the user views the related section.

The experience should feel:

* smooth
* minimal
* native-app quality
* realtime
* trustworthy

Do NOT use intrusive popup notifications for this feature.

---

# FEATURE OVERVIEW

The application already has:

* direct messages
* trip group chats
* trip post comments/replies
* authenticated users

Add unread indicators across the app.

---

# REQUIREMENTS

Implement unread tracking for:

1. Direct Messages
2. Group Trip Chats
3. Trip Post Replies/Comments

---

# UX BEHAVIOR

Example:

User receives:

* new chat message
  OR
* someone replies to their post

Then:

* a small colored unread dot/badge appears
* message tab shows unread state
* specific chat/post card shows unread indicator

Once the user opens/views the related conversation:

* unread state disappears automatically

Behavior should feel similar to:

* iPhone Messages
* Instagram DMs
* WhatsApp unread indicators
